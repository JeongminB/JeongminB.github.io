Spectral Normalization + hinge loss가 실제로 좋은지 잘 모르겠는 결과보고서 느낌의 글입니다.

---

<p>
<img src='/assets/res/dl/projection.png' width='70%' class='center'/>
</p>

(논문에 따르면) SAGAN에서는 위 그림의 projection Discriminator 구조를 사용하고, Generator는 VQA (Visual Question Answering) 문제 해결을 위한 모델에서 제안되었던 conditional BN을 사용합니다. loss function으로는 아래와 같은 hinge loss를 사용합니다.

$$ \begin{aligned}
L_D = &-\mathbb E_{x,y\sim p_{data}}[\min(0, 1-D(x,y))] \\
&- \mathbb E_{z\sim q_z,y\sim p_{data}}[\min (0, 1+D(G(z),y))] \\
L_G = &-\mathbb E_{z\sim q_z,y\sim p_{data}}[ D(G(z),y) ]
\end{aligned} $$

PyTorch로 구현하자면 아래와 같은 느낌이겠죠.

```python
def _get_d_loss(self, d_x, d_x_hat):
    if self.loss_func == 'hinge':
        return torch.mean(F.relu(1.0 - d_x)) + torch.mean(F.relu(1.0 + d_x_hat))
    elif self.loss_func == 'wgan-gp':
        return torch.mean(d_x_hat) - torch.mean(d_x)

def _get_g_loss(self, d_x_hat):
    if self.loss_func == 'hinge' or self.loss_func == 'wgan-gp':
        return -torch.mean(d_x_hat)
```

baseline을 보면 $D$에 SN을 적용했는데도 결과가 불안정하고 수렴하지 않고 있습니다 <font color='#3367E5'>[왼쪽]</font>. SN이 Discriminator를 안정화시키기 위한 기법인데 의아합니다.. mode collapse도 나타났다고 합니다.

$D$와 $G$를 1:1로 업데이트하는 것이 원인 중 하나일 수 있습니다. baseline 모델을 사용했던 논문에서는 5:1로 업데이트한 모양입니다 [Miyato et al., 2018]. SAGAN의 저자들은 1:1로 업데이트 하면서 수렴하는 모델을 찾기를 원했습니다.

SN을 $G$로 확장시키니 결과가 안정되고 나아진 듯 보이지만, 학습이 진행되면서 다시 나빠집니다. 물론 이러한 모델이 출력하는 이미지들도 사실적이지 못합니다 (질감 정도만 그럴싸한 정도). 마지막으로, $D$와 $G$의 learning rate를 다르게 가져가는 TTUR을 적용하니 1M step을 훈련하는 동안 퀄리티가 갑자기 떨어지는 일이 발생하지 않았다고 합니다.

## ◆ 결과 분석

결과적으로 그냥 SN을 하지 않고 WGAN-GP로 학습한 결과가 압도적으로 좋았습니다 (...) WGAN-GP라도 SN을 하게 되면 오히려 결과가 좋지 않았습니다. 그렇다면 왜? 라는 의문이 들기 마련입니다. 이에 대한 분석 (뇌피셜)을 적어봅니다. 

<br>

일단 논문의 조건과 많은 차이가 있습니다. SN을 제시하였던 논문의 SN-GANs이나 SAGANs이나 모두 conditional GAN입니다. $G$와 $D$의 입력으로 category 정보가 주어지기 때문에 mode dropping 문제에서 좀 더 자유로웠을 것으로 보입니다. label이 주어지는 것부터가 데이터셋을 잘게 분류를 해놓은 꼴이라? 이것의 유무는 상당히 클 것 같습니다.

그리고 사용하고 있는 가정용 GPU의 한계로 (RTX 2070 super 8GB) 모델 용량이나 미니배치 사이즈를 충분히 키우지 못하였습니다. 실험한 모델에는 모두 BN이 사용되었는데, 100개의 미니배치 샘플은 7만 장이 넘는 만화 데이터를 대표하기에 부족해보입니다. (근데 나중에 미니배치를 256으로 해도 결과는 비슷했음)

더군다나 제가 가진 데이터셋은 단일인물 뿐이지만 나름 challenging dataset이라 생각합니다. 배경, 조명(날씨 및 분위기), 명암, 화풍(그림체), 색상(머리, 눈, 피부 등), 구도(정면, 측면 외 다양한 각도, 단순 회전), 크기(이미지 내에서 인물이 차지하는 비중), 표정, 스타일(헤어, 장신구, 옷), 화질, 디테일(굵고 단순한 그림, 얇고 세밀한 그림) 등 굉장히 다양한 (자연적이지 않은) 변수가 있기 때문이죠.

<br>

또 다른 추측으로는, SN 방식과 GP 방식이 잘 어울리지 않을 수 있다는 점입니다. SN을 하면 largest singular value를 추정하게 되는데 iteration이 1이면 추정오차가 좀 있습니다. GAN 자체가 non-stationary한데 여기에 매 step마다 SN에 의한 scailing이 흔들리면 수렴시키기 어려운 요소가 늘어나는게 아닌가 싶습니다. (물론 단순히 정확도를 늘린다고 WGAN-GP 성능을 상회할 것 같진 않습니다만)

맨 위에 제시했던 그래프에서 baseline 모델이 불안정한 것도 이와 어느 정도 관련이 있지 않나.. 사실 baseline이 너무 구린 것도 좀 그렇지만 저는 그러한 불안정한 상태에서 $G$로 SN을 확장시켰다고 학습 곡선이 어느 정도 안정화 되었다는 사실을 받아들이기 어렵네요. TTUR을 적용했을 때 결과가 크게 안정화된 것도 그렇구요.

<p>
<img src='/assets/res/etc/sagan_result2.jpg' width='70%' class='center'/>
</p>

Spectral Normalization 논문에서는 오히려 WGAN-GP가 높은 learning rate에서 결과가 좋지 않으며, SN을 적용한 SN-GANs는 이러한 하리퍼파라미터에 변화에 robust하다고 주장하고 있기 때문에 뭔가 이야기가 맞지 않습니다 -_-;;

마지막으로 아까 얘기한 scailing의 연장선인데, GANs이 non-stationary하다는 점에서 largest singular value를 갖는 (모델의 각 layer weight matrix에 해당하는) 선형변환의 축? 역시 계속해서 변할 수 있지 않나 싶습니다.

Spectral Norm.은 matrix 변환에 의한 벡터 L2 norm scale의 sup (max)값이므로 이것이 정확히 1이 되도록 레이어마다 맞추는 작업 자체가 landscape를 불안정하게 하지 않나.. (추측임).

함수의 Lipschitz constant를 1이 넘지 않게 제한하는 것도 여러 가정 하에 Wasserstein-1 distance를 어디까지나 '근사'할 수 있다는 것이지 ($D$ 반복 학습을 통해), 그러한 제약으로 optimal $D$를 만든다는 보장은 없습니다. 같은 loss에 GP term만 추가하면 optimal $D$의 특성을 유도할 수 있으나, 어쨌든 결과가 좋지 않았습니다.

stationary한 문제에서는 SN이 빛을 발휘할 수 있을지도 모릅니다. 아니면 제가 캐치하지 못한 부분이 있다든지, 가능성은 낮겠지만 PyTorch에서 제공하는 `spectral_norm()` 자체의 문제나 그것의 오용 문제일수도 있겠구요.

<br>

hinge loss에 대한 논의가 없었는데 저는 이것을 사용하는 이유, 장점에 대해서 특별히 알아내지 못하였습니다. hinge loss는 직관적으로, $D$가 real에 1이상의 출력을 내면 loss가 0이고 fake는 -1하의 출력을 내면 loss가 0인데 $G$는 $D$의 출력을 높이도록 fake를 학습하게 됩니다.

`spectral_norm()`의 실제 동작을 (1) 글에서 서술한 것처럼 직접 테스트해보며 확인했기 때문에 따로 구현해보거나 그러지는 않을 예정입니다. 만약 세 번째 글을 쓰게 된다면 CBN과 projection Discriminator 까지 구현해서, 지도학습 결과물을 가져와야할 것 같네요. 그러면 좀 더 이야기가 확실해질 것 같습니다.

결론 : 적어도 비지도에서는 WGAN-GP가 짱인 듯?
