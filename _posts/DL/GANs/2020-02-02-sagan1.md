SAGAN은 conditional image generation on ImageNet에서 SOTA를 달성했던 모델입니다. 논문에서는 자세한 GAN 구조를 제시하고 있지 않습니다. 이 글은 SAGAN에서 conditional한 요소를 제외하고 PyTorch로 구현하면서 정리한 글입니다.

## ◆ spectral_norm()

PyTorch에서는 power iteration method를 통한 `spectral_norm`이 이미 구현되어 있습니다. `spectral_norm`은 module을 받아, SN을 적용하는 forward pre-hook을 걸은 module을 반환합니다. 모듈의 forward가 호출되기 전에 hook이 수행되는데, spectral norm이 계산되어 weight를 scaling 하는 역할을 합니다.

```python
class Test(nn.Module):
    def __init__(self, n_piter=1):
        super().__init__()
        self.layer = spectral_norm(nn.Linear(2,2, bias=False), n_power_iterations=n_piter)

    def forward(self, x):
        return self.layer(x)
```

예를 들면, 위처럼 `Linear`나 `Conv` 레이어 따위를 감싸주면 됩니다.

```python
x = torch.rand([1, 2])
model = Test(n_piter=1)
_ = model(x)

print(model.layer.weight_orig)
print(model.layer.weight)
_,s,_ = torch.svd(model.layer.weight_orig, compute_uv=False)
_,s_new,_ = torch.svd(model.layer.weight, compute_uv=False)
orig_sn = s[0].item()
updated_sn_by_PIM = s_new[0].item()
print(f'orig_sn: {orig_sn}')
print(f'updated_sn_by_PIM: {updated_sn_by_PIM}')
```

`spectral_norm()`을 쓸 경우, $W$는 `weight_orig`와 `weight` 이렇게 나뉘게 됩니다 (일반적인 모듈은 `weight_orig`가 없음). `weight_orig`를 자신의 spectral norm $\hat\sigma_1$로 나누어주면 `weight`가 됩니다. 물론, input $\boldsymbol h$는 `weight` $\bar W_{\text{SN}}(W)$와 연산을 수행합니다.

다만, `optim.step()` 이후에 `weight_orig`에 `weight`가 복사되거나 그러지는 않습니다. 즉, 실제 update 대상은 `weight_orig`이고, 추가로 scailing을 담당하는 파트가 추가되어 있을 뿐입니다. `spectral_norm`의 `n_power_iterations=`이 1이면 어느 정도 추정 오차가 있으며, 2정도만 되어도 layer의 Lipschitz norm이 거의 1에 근접하게 됩니다. 이것이 유의미한 성능 차이를 만들어내는지는 모델에 따라 직접 확인해봐야 할 것 같네요.

## ◆ non-local block

<p>
<img src='/assets/res/dl/nlnn.png' width='70%' class='center'/>
</p>

$$ \mathbf {y}_i =  \underbrace{\frac{1}{C(\mathbf x)}}_{\mathrm{norm.}} \sum_{\forall j} \underbrace{f(\mathbf{x}_i, \mathbf{x}_j)}_{\mathrm{similarity}} \; \underbrace {g(\mathbf{x}_j)}_{= W_g \mathbf{x}_j} $$

위 그림은 video input 기준이고 <a href="/dl/gans/nlnn/#-non-local-neural-networks">*"Non-local Neural Networks"*</a> 논문에서 사용된 그림입니다. 위와 같은 구조는 국소적인 영역밖에 못 보는 conv layer의 한계점을 해결해줄 수 있습니다. 위 식은 Non-local operation으로, 간단히 말하면 두 대상의 유사도를 weight로 해서 sum을 구하는 연산입니다.

$$ % <![CDATA[
\begin{aligned}
{}_\text{Gaussian} &: f(\mathbf{x}_i, \mathbf{x}_j) = e^{\mathbf{x}_i^T \mathbf{x}_j} \\
{}_\text{Embedded Gaussian} &: f(\mathbf{x}_i, \mathbf{x}_j) = e^{\theta(\mathbf{x}_i)^T \phi(\mathbf{x}_j)} \\
{}_\text{Dot product} &: f(\mathbf{x}_i, \mathbf{x}_j) = \theta(\mathbf{x}_i)^T \phi(\mathbf{x}_j) \\
{}_\text{Concatenation} &: f(\mathbf{x}_i, \mathbf{x}_j) = \mathrm{ReLU}(\mathbf{W}_f^T[\theta(\mathbf{x}_i), \phi(\mathbf{x}_j)]) 
\end{aligned} %]]> $$

이때 유사도를 구하는 식을 Embedded Gaussian으로 하고, 모든 입력을 동일한 Tensor로 바꾸면 Transformer 모델을 구성하는 Self-attention과 같은 꼴이 됩니다. Non-local 연산의 norm. 부분과 Self-attention의 dot-product를 $1/\sqrt{d_k}$로 스케일링하는 부분을 빼면 말이죠.

$$ \boldsymbol y_i = \gamma \boldsymbol o_i + \boldsymbol x_i $$

SAGAN 논문에서는 Self-attention 출력 $\boldsymbol o$에 초기값이 0인 parameter $\gamma$를 곱하고 identity $\boldsymbol x$를 shortcut으로 연결한 모듈을 제안하고 있습니다. 처음에는 $x$를 그대로 출력하기 때문에 일반적인 CNN 모델과 비슷하고, 점차 long-range dependency를 배우게 하려는 의도인 것 같습니다.

<br>

module을 구현하는 법을 알아봅시다.

* $Q,K,V$ : `[C, H, W]` $\rightarrow$ `[C/k, H, W]` $\rightarrow$ `[C/k, HW]`
* $Q^{\text T}K$ : `[HW, HW]`
* $V\cdot\text{softmax}(Q^{\text T}K)^{\text T}$ : `[C/k, HW]` $\rightarrow$ `[C/k, H, W]` $\rightarrow$ `[C, H, W]`

PyTorch 방식 기준으로 위와 같은 연산 및 size를 따라야 합니다 (논문과 표기 차이가 있습니다). 채널을 `c`에서 `c/k`로 줄이는 이유는, 이렇게 해도 성능 저하가 일어나지 않았다는 이유 때문인 듯.

이미지의 경우 1x1 conv로 channel reduction / expansion 을 구현하고, 나머지는 matrix multiplication과 transpose만 잘 처리해주면 되겠습니다. transpose는 `.permute()`로 구현합니다.

```python
def __init__(self, in_ch, div_num=8):
    super().__init__()
    assert in_ch % div_num == 0, "The remainder of 'in_ch/div_num' must be zero."

    self.q_conv1x1 = spectral_norm( nn.Conv2d(in_ch, in_ch//div_num, kernel_size=1)
                        , n_power_iterations=n_power_iter)
    self.k_conv1x1 = spectral_norm( nn.Conv2d(in_ch, in_ch//div_num, kernel_size=1)
                        , n_power_iterations=n_power_iter)
    self.v_conv1x1 = spectral_norm( nn.Conv2d(in_ch, in_ch//div_num, kernel_size=1)
                        , n_power_iterations=n_power_iter)
    self.sa_conv1x1 = spectral_norm( nn.Conv2d(in_ch//div_num, in_ch, kernel_size=1)
                        , n_power_iterations=n_power_iter)
    self.gamma = nn.Parameter(torch.zeros(1))
```

논문에서는 spectral normalization을 모든 레이어에 적용했다고 말하고 있습니다. 논문대로라면 위와 같이 레이어들을 준비해야할 것 같네요.

```python
def forward(self, x):
    batch_size, c, h, w = x.size()
    q = self.q_conv1x1(x).view(batch_size, -1, h*w)
    k = self.k_conv1x1(x).view(batch_size, -1, h*w)
    v = self.v_conv1x1(x).view(batch_size, -1, h*w)

    attn_map = F.softmax(torch.bmm(q.permute(0,2,1), k), dim=-1)
    sa_map = torch.bmm(v, attn_map.permute(0,2,1))
    sa_map = self.sa_conv1x1(sa_map.view(batch_size, -1, h, w))
    return self.gamma*sa_map + x
```

forward pass는 위와 같이 작성할 수 있겠습니다. `.permute()`로 $Q$를 `[(B,) HW, C]` 형태로 바꿔주고, $Q^{\text T}K$ 연산을 batch마다 수행합니다. 그리고 마지막 차원에 대해 softmax를 적용합니다.

## ◆ non-local block (with max pooling)

또 다른 형태의 non-local block을 제시합니다. 구조 자체는 동일하지만 중간에 pooling을 도입합니다. 논문 저자들이 공개한 코드에는 이러한 형태의 non-local block을 쓰고 있는데, 논문에서는 소개되지 않은 부분이라 실제 실험에 썼다고 보장할 수는 없을 듯.

<p>
<img src='/assets/res/dl/nlb.png' width='80%' class='center'/>
</p>

풀링은 Max Pooling을 사용합니다. Average Pooling과 달리 Max Pooling은 translation invariance을 가지고 있고 비선형 연산입니다.

개인적인 생각인데 feature map은 대상의 spatial 정보를 담고 있을 것이고, Max Pooling은 kernel의 max 값을 뽑아 내기 때문에, feature map에서 기여도가 적은 부분 (noise)을 늘리는 꼴이 되지 않을까.. (overfitting 개선가능?)

```python
def forward(self, x):
    batch_size, c, h, w = x.size()
    HW = h*w
    q = self.q_conv1x1(x).view(batch_size, -1, HW)
    k = self.k_conv1x1(x)
    v = self.v_conv1x1(x)
    if self.maxpool:
        k = F.max_pool2d(k, (2,2))
        v = F.max_pool2d(v, (2,2))
        HW = HW // 4

    attn_map = F.softmax(torch.bmm(q.permute(0,2,1), k.view(batch_size, -1, HW)), dim=-1)
    sa_map = torch.bmm(v.view(batch_size, -1, HW), attn_map.permute(0,2,1))
    sa_map = self.sa_conv1x1(sa_map.view(batch_size, -1, h, w))
    return self.gamma*sa_map + x
```

어쨌든 단순하게 짜보았습니다. `__init__` 부분만 살짝 수정하면 될 것 같네요. 위처럼 구현할 경우 아래에 있는 HW가 위에 있는 HW와 다르다는 점에 유의합니다.

## ◆ Generator

<p>
<img src='/assets/res/dl/sagan3.png' width='100%' class='center'/>
</p>

논문에는 없지만 $G$의 가능한 예시를 소개합니다.

일반적인 DCGAN처럼 채널 수를 줄여가면서 upscale을 하고, 마지막에 BN ReLU Conv2d tanh 를 통과한 후에 fake image $\hat x$을 얻게 됩니다. 모델에 쓰인 모든 Conv, FC 레이어에는 SN을 적용합니다.

아래에서 소개할 Upscaling Block을 총 5개 쓰며, 블록들 사이에 Self-Attention Block을 끼워넣게 됩니다.

<p>
<img src='/assets/res/dl/sagan4.png' width='70%' class='center'/>
</p>

Upscaling Block의 구조입니다. shortcut에서는 업스케일 결과와 입력 $\boldsymbol x$를 합칠 수 있도록 스케일링과 채널 매칭만 이루어집니다. 논문의 모델에서는 BN 대신 CBN이 사용되었지만 넘어가겠습니다. upscaling 방식은 nearest neighbor 알고리즘을 활용하겠습니다.

<br>

<p>
<img src='/assets/res/dl/sagan2.png' width='100%' class='center'/>
</p>

표는 non-local block의 위치를 바꾸어가며 성능을 테스트한 결과입니다. 굳이 하나만 넣어야 한다는 이야기는 없지만, 블록 하나로도 conditional image generation on ImageNet SOTA를 달성했다고 하네요.

표에서 $feat_{8}$의 의미는 feature map의 크기가 `8x8`인 위치에 block을 넣었다는 이야기입니다. $feat_{32}$ 혹은 $feat_{64}$ 정도가 `128x128` output 구조에서 적당한 위치인 것 같습니다. 그리고 $D$에서도 같은 위치에 non-local block을 삽입합니다.

오른쪽은 non-local block을 같은 파라미터 수를 갖는 residual block으로 교체했을 때의 결과입니다. $feat_{8}$에서의 성능이 크게 떨어지는 점이 돋보이네요. 표를 통해서 모델의 용량과 깊이가 성능을 크게 좌우하는 것은 아니라는 점도 확인할 수 있습니다.

## ◆ Discriminator

<p>
<img src='/assets/res/dl/sagan5.png' width='90%' class='center'/>
</p>

$D$는 $G$와 거의 대칭 구조를 이룹니다. 원래는 projection Discriminator를 쓰기 때문에, $x$에 대응하는 label 벡터를 `df*16` 사이즈로 embedding하여, reduce_sum을 한 출력과 mm한 후에 output에 더해주는 작업이 필요하나 생략했습니다.

<p>
<img src='/assets/res/dl/sagan6.png' width='70%' class='center'/>
</p>

$D$에 들어가는 Downscaling Block 구조입니다. 맨 처음 블록 입력으로는 이미 normalizing 된 입력이 들어가기 때문에 맨 앞의 activation을 생략하고, shortcut의 Conv 와 Downsample 위치를 바꿔줍니다.

그 이후의 블록에서는 위의 구조를 따르며, 마지막 블록에서는 Downsample을 하지 않도록 class를 구현합니다. Downsampling은 `nn.AvgPool2d()`을 사용합니다.

## ◆ implementation details

논문에 있는 구현 세부 사항을 종합해보면 다음과 같습니다.

* **Generator** + CBN + Spectral Normalization
* **Discriminator** + projection (label embedding) +SN
* **loss** : hinge loss (see the paper)
* **mini-batch size** : 256 (64 x 4)
* **optimizer** : Adam $(\beta_1=0, \beta_2=0.9)$
* **lr_g** : 0.0001  (TTUR)
* **lr_d** : 0.0004  (TTUR)
* **update ratio (D:G)** : 1:1 (balanced)
* **image size** : 128 x 128

(2)에서는 결과에 대한 이야기를 짤막하게 나눠볼까 합니다.
