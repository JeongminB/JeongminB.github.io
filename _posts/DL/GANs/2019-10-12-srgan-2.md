ILSVRC-2015 우승모델 **ResNet**은 152층 모델입니다. 그리고 모델 성능은 오류율만 놓고 보면 인간의 인지 능력을 넘은 것으로 평가되었죠. 깊은 모델이 성능이 좋다는 것을 보인 대표적인 예가 아닌가 싶습니다. 그러나 마냥 깊기만 해서는 끝 쪽의 layer에서 training이 잘 이루어지지 않는 <strong><font color='#FF7D33'>degradation</font></strong> 문제가 발생합니다. 이에 ResNet은 skip connection 방식을 이용하는 Residual Block을 만들어서 신경망을 구성하였습니다. 이러한 구조는 SRResNet, SRGAN에서도 사용되고 있습니다.

이후에 나온 **Wide ResNet** 논문에서는 ResNet의 Block을 변화시켜가며 성능 변화를 관찰하였습니다. 논문에 따르면 convolution은 (3 x 3) 구조가 성능이 뛰어났고, 한 Residual Block 내에 convolution 층 수는 2개가 제일 좋았다고 합니다. 그리고 층을 줄이고 width를 늘리는 방식이 모델의 용량은 커져도 더 나은 성능과 학습 능력을 보여주었습니다. 이것은 trade-off가 있어서 최적의 하이퍼파라미터를 찾으려면 실험적으로 접근해야할 것 같습니다.

이 글에서는 직접 모델을 구현해보면서 겪은 문제에 대해 제멋대로 고찰하는 시간을 가져보겠습니다.

---

 처음으로 SRGAN을 구현한 후 학습시킬 때 지인으로부터 운 좋게 구하게 된 cartoon data를 사용했습니다. 제가 생각해본 만화 데이터만의 장점은 아래와 같습니다.

 * 색 분포가 고르면서 그라데이션인 형태가 많아 육안으로 퀄리티를 확인하기 쉬움
 * 경계가 뚜렷하고 값의 차이가 크기 때문에 high frequency 표현 성능을 확인하기 좋음
 * 많은 양의 고화질 데이터를 얻기에 용이함

확실하진 않지만 저는 ImageNet에 만화 데이터가 없는 걸로 알고 있습니다. 여기서 한 가지 관심사항은 ImageNet에 대해 훈련된 VGG모델을 만화 데이터에 적용했을 때 좋은 결과가 나오느냐였죠. 아무래도 일반 이미지와 만화 이미지의 특성이 차이가 많기 때문입니다.

#### ◆ 학습 데이터 처리

맨 처음 학습에서 loss는 VGG22의 feature map에 대한 MSE를 content loss로 하여 계산하였습니다. 아래에 있는 출력물들은 학습 과정을 보인 것이며, model evaluation은 따로 다루겠습니다.

<br>

<p>
<span class='center'><img src='/assets/res/gans/srgan-img5.jpg' width='40%' /> &nbsp; <img src='/assets/res/gans/srgan-img6.jpg' width='40%' /></span>
<span class='center'>srgan-test1</span>
</p>

학습의 진행은 직접 샘플을 출력해보면서 확인하였습니다. 왼쪽 세트가 3500장 정도 학습한 학습 초기, 오른쪽 세트가 15만장 (2 epochs 정도) 정도 학습한 상황입니다. 그리고 각각의 세트에서 왼쪽이 저해상도 (64 x 64), 오른쪽이 고해상도 (128 x 128), 가운데가 SRGAN의 Generator가 내놓은 이미지입니다. 저해상도 데이터는 **nearest neighbor** 방식으로 업스케일하였습니다. `nn.functional.interpolate()`의 default 값입니다.

제일 먼저 발견된 문제점은 출력 이미지의 색상이 전체적으로 변화한다는 것입니다. 그리고 오른쪽의 첫 번째 결과에서 살짝 나타나는데, 어두운 사진에서 테두리 주위에 빛 효과가 생기는 기현상도 관찰되었습니다.

```python
def forward(self, d_fake, real, fake):
    content_loss = self.content_loss(real, fake)
    adv_loss = torch.mean(-torch.log(d_fake + self.epsilon))
    return content_loss + self.w*adv_loss
```

loss의 계산은 위와 같이 class 내에서 이루어졌고, `self.w=1e-3`으로 논문에서 소개된 조건을 이용하였습니다. content loss function인 VGG22는 parameter들의 `requires_grad`를 False로, 모듈에는 `.eval()` 메소드를 불러서 VGG22 layer의 학습 및 불필요한 계산을 막아놓았습니다. 색감 변화의 원인을 알기 위해 테스트 조건을 바꿔보기로 했습니다.

<br>

<p>
<span class='center'><img src='/assets/res/gans/srgan-img7.jpg' width='40%' /> &nbsp; <img src='/assets/res/gans/srgan-img8.jpg' width='40%' /></span>
<span class='center'>srgan-test2 &nbsp; &nbsp; srgan-test3</span>
</p>

VGG22 대신 MSE를 content loss로 쓴 결과입니다. 역시나 색감이 달라지는데 이번에는 굉장히 어두운 이미지들을 내놓았습니다. 학습이 꽤 진행되어도 문제는 계속되었습니다 (왼쪽 그림).

원인은 data를 0~1 사이로 놓는 `ToTensor()`을 거친 후 Standardization을 멋대로 한 것 때문이었습니다. 모델을 구성할 때 generator가 마지막에 tanh를 거치지만 최종 출력은 다시 [0, 1]로 바꾸는 바람에... 원인을 알고 나니까 맨 처음 결과가 그럴싸하게 나온 것이 신기할 정도네요. transforms에서 `normalization()`을 모두 없애주니 원본의 색감과 비슷한 결과를 내게 되었습니다 (오른쪽 그림). 위의 두 그림은 모두 3500장 정도 학습한 상황입니다.

확실히 알 수 있는 건 VGG 모델을 불러와 활용했을 때의 성능이 월등히 좋다는 것입니다. 복잡한 이미지들을 구분할 수 있도록 이미지의 특성을 분해하는 능력이 뛰어난 덕분이 아닌가 싶네요. '일반적인' convolution이나 batch normalization layer의 가중치 초기화는 결과에 크게 영향을 미치지 않았습니다. 그리고 VGG의 input으로 집어 넣을 때 흔히 하는 `mean=(0.485, 0.456, 0.406)`, `std=(0.229, 0.224, 0.225)` standardization도 별 다른 개선점은 없었습니다.

이때부터 이미지 resize를 LANCZOS 방식으로 바꿨습니다. 원본 데이터에서 훈련용 데이터셋을 만들 때 (256 x 256) 사이즈로 줄이는 과정에서 LANCZOS 방식을, SRGAN 훈련 과정에서 저해상도의 입력 이미지를 만들 때에도 같은 방식을 이용하였습니다.

#### ◆ Checkerboard artifact

훈련을 통해서 해결하지 못한 문제가 두 개 있는데, 하나는 약간의 색상 변화, 다른 하나는 격자 형태의 noise나 얼룩이 출력 이미지에서 나타나는 것입니다. 여기서 후자를 흔히 <strong><font color="#3367E5">Checkerboard artifact</font></strong>라고 부릅니다.

<strong>Transposed convolution</strong>으로 이미지를 키운 경우, kernel size가 stride로 나누어떨어지지 않으면 합성곱 계산 과정에서 filter가 overlap 되는 구간이 생겨 이러한 현상이 나타납니다.

<p>
<img src='/assets/res/gans/srgan-img3.png' width='80%' class='center'/>
<span class='center'>출처: https://arxiv.org/abs/1707.02937</span>
</p>

<strong>Sub-pixel convolution</strong>은 원래 이미지와 같은 크기의 특성맵을 `upscale_factor ** 2`만큼 내놓게 합성곱 신경망을 구성하여, 이들을 섞어주는 식으로 출력의 size를 늘립니다. 각각 다르게 초기화된 필터들이 나누어지면서 checkerboard artifact가 생기는 것 같습니다. 물론 학습을 많이 진행하면 문제가 해결되겠지만, sub-pixel convolution의 경우 필터의 초기화 방법을 바꾸어주면 이 문제를 해결할 수 있습니다.

<strong><font color="#3367E5">ICNR</font></strong> (initialized to convolution NN resize) 이라는 방법을 쓰면 됩니다. 원리는 `upscale_factor ** 2` 단위로 커널 초기화를 통일시켜주는 것입니다. 위 그림으로 설명하면 `(4,1,2,2)` 크기의 가중치 초기화를 `(1,1,2,2)`를 4번 반복한 것으로 하자는 것이죠. (구현법은 생략..)

<br>

<p>
<span class='center'><img src='/assets/res/gans/srgan-img10.jpg' width='40%' /> &nbsp; <img src='/assets/res/gans/srgan-img9.jpg' width='40%' /></span>
<span class='center'>srgan-test4 &nbsp; &nbsp; srgan-test5</span>
</p>

학습 극초반의 결과물을 가져왔습니다. ICNR을 적용한 후가 오른쪽인데, 왼쪽에서 나타나는 촘촘한 checkerboard 패턴이 없는 것을 확인할 수 있습니다.

#### ◆ Scale factor of 2 vs 4

<p>
<span class='center'><img src='/assets/res/gans/srgan-img11.jpg' width='40%' /> &nbsp; <img src='/assets/res/gans/srgan-img12.jpg' width='40%' /></span>

</p>

이번에는 upscale factor를 2와 4로 한 경우입니다. 양쪽 다 15만장 정도 학습한 상황입니다. 크롭한 원본 이미지의 크기는 (128 x 128)이며, Generator 입력으로 들어가는 저해상도 이미지 크기는 각각 (64 x 64), (32 x 32) 입니다. 보통 SRGAN에서는 upscale factor를 4로 놓고 학습을 시키는데, 만화 이미지에서는 중요한 정보들이 날아가면서 디테일을 살리진 못하고 있습니다.
