```python
torch.nn.Linear(in_features, out_features, bias=True)
```

$$ y = x A^T + b $$

`Linear` 레이어는 TensorFlow 혹은 Keras의 `Dense` 레이어에 해당하는 익숙한 레이어입니다. 가중치 `Linear.weight`는 $(\text{out_features}, \text{in_features})$ 의 shape을 가지며, shape이 $(N,\ast,\text{in_features})$ 인 input과 선형 변환 연산을 수행합니다. $N$은 batch size를 의미하고, $*$ 는 데이터의 마지막 차원을 제외한 나머지 차원을 의미합니다. `Linear.bias` $b$는 shape이 $(\text{out_features})$ 이고, 행렬 곱 연산 결과에 더해집니다. 필요시에 bias는 broadcast 됩니다.

이 연산의 결과로 얻는 Tensor의 shape은 $(N,\ast,\text{out_features})$ 입니다. 즉, 선형 변환 결과로 영향을 받을 수 있는 차원은 마지막 차원 뿐이고 나머지는 언제나 동일한 차원을 갖습니다.

## ◆ torch.nn.Bilinear()

```python
torch.nn.Bilinear(in1_features, in2_features, out_features, bias=True)
```

$$ y = x_1 A x_2 + b $$

PyTorch의 Linear layers에는 `Linear` 레이어 외에도 `Bilinear` 레이어가 있습니다. `Bilinear`는 입력 데이터에 대한 위와 같은 쌍선형 변환 (bilinear transformation)을 한다고 나와있습니다. DSP (Digital Signal Processing) 에서의 쌍선형 변환은 Laplace transform에 의한 시스템의 s-domain transfer function을 z-domain으로 변환하는 방법을 말합니다만, 당연히도 이 글에서는 무관합니다.

위 식에서 각 Tensor의 shape을 알아봅시다. 먼저 bias는 `(out_features)` 형태입니다. 가중치에 해당하는 $A$는 `(out_features, in1_features, in2_features)` 이렇게 3차원 형태입니다. 그리고 각 입력은 `(N, *, in1_features)`, `(N, *, in2_features)` 형태입니다. 이들의 곱셈이 어떻게 이루어질까요? 뭐니 뭐니 해도 예시를 드는 편이 제일 좋을 것 같네요.

## ◆ bilinear 연산의 이해

```python
import torch
import torch.nn.functional as F

b  = [0]                      # [1]
x1 = [1, 1]                   # [2]
x2 = [1, 1, 1]                # [3]
A  = [[[1, 2, 3],[1, 2, 3]]]  # [1 2 3]

x1 = torch.tensor(x1)
x2 = torch.tensor(x2)
A = torch.tensor(A)
b = torch.tensor(b)

print(b.size(), x1.size(), x2.size(), A.size())
y = F.bilinear(x1, x2, A, b)
print('',  y.size(), '\n' , out)
```

```
torch.Size([1]) torch.Size([2]) torch.Size([3]) torch.Size([1, 2, 3])
 torch.Size([1]) 
 tensor([12])
```

위와 같이 $A$를 2차원 matrix가 쌓인 형태로 취급하여 두 입력에 대한 각각의 연산을 수행합니다. 그리고 각 연산의 결과가 출력 $y$의 요소가 됩니다. 코드로 확인해보면 아래와 같습니다.

```python
import torch
import torch.nn.functional as F

b  = [0, 0, 0, 1]             # [4]
x1 = [2, 1]                   # [2]
x2 = [1, 1, 1]                # [3]
A  = [[[1, 2, 3],[1, 2, 3]],
      [[2, 3, 4],[2, 3, 4]],
      [[0, 0, 0],[0, 0, 9]],
      [[1, 2, 3],[1, 2, 3]]]  # [4 2 3]

x1 = torch.tensor(x1)
x2 = torch.tensor(x2)
A = torch.tensor(A)
b = torch.tensor(b)

print(b.size(), x1.size(), x2.size(), A.size())
y = F.bilinear(x1, x2, A, b)
print('',  y.size(), '\n' , y)
```

```
torch.Size([4]) torch.Size([2]) torch.Size([3]) torch.Size([4, 2, 3])
 torch.Size([4]) 
 tensor([18, 27,  9, 19])
```

각각의 값이 어떻게 나왔는지 이해되지 않는다면 직접 계산을 따라가 보는 것도 좋을 것 같습니다. 두 입력 벡터가 `Bilinear` 레이어를 통과하면 scalar 혹은 scalar 값을 여러 개 이은 vector를 내놓는다는 점을 기억하세요.

## ◆ 입력이 복잡한 형태의 경우

미니배치 방식으로 학습시키기만 해도 입력은 `[N, features]` 꼴입니다. 더 나아가 입력이 `[N, 3, ... , 5, 10]` 같은 형태일 수도 있겠죠. bilinear 연산은 $A$의 shape을 출력과 두 입력의 마지막 차원으로 맞추고, 두 입력이 마지막 차원을 제외하고 같은 shape을 가진다면 상관없습니다. 역시 최대한 간단한 예를 들어보겠습니다.

```python
b  = [0]                       # '1'                     
x1 = [[[1, 1], [1, 1]],
      [[1, 1], [1, 1]]]        #  2  2 '2'
x2 = [[[2, 1, 1], [1, 1, 1]],
      [[1, 1, 1], [2, 1, 1]]]  #  2  2 '3'
A  = [[[1, 2 ,3],[4, 5, 6]]]   #  1  2  3

x1 = torch.tensor(x1)
x2 = torch.tensor(x2)
A = torch.tensor(A)
b = torch.tensor(b)

print(b.size(), x1.size(), x2.size(), A.size())
y = F.bilinear(x1, x2, A, b)   #  2  2 '1'
print('',  y.size(), '\n' , y)
```

```
torch.Size([1]) torch.Size([2, 2, 2]) torch.Size([2, 2, 3]) torch.Size([1, 2, 3])
 torch.Size([2, 2, 1]) 
 tensor([[[26],
         [21]],

        [[21],
         [26]]])
```

입력이 `2,2,in1_features`, `2,2,in2_features` 꼴로 바뀌어도, 계산 과정은 마치 길이가 `in1_features`, `in2_features`인 벡터 두 개가 입력인 'bilinear 연산들의 모임'과 동일합니다.

즉, 벡터가 `2 x 2`개 만큼 존재하며, 이들의 연산이 각각 이루어져 `out_features` 길이의 벡터들의 모임인 `2,2,out_features` 형태의 출력을 얻는다고 생각하면 좋을 것 같습니다. 입력이 복잡한 형태의 `Linear` 연산도 위와 같은 방식으로 이루어집니다.

이다음에 다루는 내용은 사실 따로 작성해야 하는 별개의 글입니다 (나중에 더 자세히 다루게 될 경우 글을 분리시킬 수도 있습니다). `Bilinear` 레이어가 어떤 의미를 가질 수 있는지, 어떻게 활용될 수 있는지 보충 설명하기 위한 용도로 논문 하나를 짧게 훑어보았습니다. 이해가 완벽하신 분들은 굳이 읽으실 필요는 없겠네요.

## ◆ Reasoning With Neural Tensor Networks for Knowledge Base Completion

`Bilinear` 레이어가 구체적으로 어떻게 활용될 수 있는지 알아보기 위해, <strong><font color='#3367E5'>NTN</font></strong> (neural tensor network) model을 소개하고자 합니다. 이는 자연어 처리 에서 Knowledge Base Completion 작업을 위한 추론 모델을 구현하는데 쓰였습니다.

Natural Language Processing 에서 지식 기반 (knowledge bases)은 대표적으로 질의 응답 (question answering) 시스템에 활용될 수 있습니다. 그러나 이산 개체들 (discrete entities)과 이들 사이의 관계를 추론하는 능력이 부족하거나 불완전한 어려움을 겪곤 합니다.

잠깐 용어에 대해서 알아보겠습니다. NLP 영역에서는 주로, 특정 도메인에서 수집된 자연어 데이터를 <strong><font color='#FF7D33'>corpus</font></strong> (말뭉치)라고 부릅니다. <strong><font color='#FF7D33'>entity</font></strong>는 말 그대로 개체입니다. 참고로, `'Google'`, `'2019'`, `'Seoul'` 같이 고유한 이름이 붙은 개체를 `'ORG'`, `'DATE'`, `'LOC'`와 같이 분류하는 방법론을 개체명 인식 <strong><font color='#FF7D33'>NER</font></strong> (Named-entity recognition)이라고 합니다. 

Richard Socher이 논문(2013)에서 제시한 지식 기반 완성 모델의 목표는, 기존의 지식 기반 (knowledge base)을 토대로 추가적인 사실을 예측하는 것입니다. 예를 들어, '○○원숭이' 라는 새로운 원숭이 종(種)이 발견되었다고 합시다. '○○원숭이는 다리가 있나요?' 라는 기존에 없던 추가 질문에 대한 답을 내리기 위해 사람은 굳이 다른 글을 참고할 필요가 없지요. '원숭이는 다리가 있다'라는 지식이 있기 때문에, 새로운 원숭이도 다리가 있을 것이라고 '예측'할 수 있기 때문입니다.

이를 모델로 구현하기 위해서는 신경망 모델이 각 entity 간의 관계성, 상호작용을 이해하고 있어야 합니다. 이를 위해 각각의 entity를 vector로 나타냅니다. 벡터로 나타내면 entity에 대한 사실이나, entity가 특정 <strong>관계</strong>에 포함될 가능성을 알아낼 수 있을 겁니다. 방법론에 대해서는 곧 소개하겠습니다.

여기서 <strong>'관계'</strong> $R$ (relation)은 두 entity vector를 관계시킬 수 있는 '신경망 파라미터를 통해 정의'됩니다. 논문에서 제시된 네 가지 관계 모델 중에서 single layer model, bilinear model 두 개만 소개하고자 합니다.

### ◇ Single Layer Model

$$ g(e_1, R, e_2) = u_R^Tf \left([W_{R,1} \; W_{R,2}] \begin{bmatrix} e_1 \\ e_2 \end{bmatrix} \right) $$

$f=\tanh$, $W_{R,1}$, $W_{R,2}\in \mathbb{R}^{k\times d}$, 그리고 $u_R \in \mathbb{R}^{k\times 1}$ 일 떄, 두 entity $e_1, e_2 \in \mathbb{R}^d$ 의 상호작용 score를 계산할 수 있습니다. 이러한 구조는 비선형성을 이용해 두 entity를 암시적으로 결합하기 때문에 단순 distance model 보다는 낫지만, 최적화가 어려운 문제에서는 약한 상호작용 (weak interaction)을 가지게 됩니다.

### ◇ Bilinear Model

$$ g(e_1, R, e_2) = e_1^TW_Re_2\;, \quad \text{where } W_R \in \mathbb{R}^{d\times d} $$

Bilinear model은 약한 상호작용 문제를 관계별 (relation-speciﬁc) 쌍선형 형식 (bilinear form)을 통해서 해결합니다. 보다시피 단순하지만 직접 두 entity vector를 결합하여 score를 계산할 수 있습니다.

그러나, 임베딩된 단어 벡터의 크기에 의해 모델의 파라미터의 수나 표현력에 제한을 받고 있습니다. 또한 위와 같은 형태로는 linear interactions 만 모델링하게 되며, 보다 복잡한 scoring function을 구성할 수 없네요. 그래서 둘의 장점을 결합한듯한 좀 더 일반화된 NTN model을 제시합니다.

## ◆ Neural Tensor Network Model

<p>
<span class='center'>
<img src='/assets/res/etc/ntn1.png' width='70%' /></span>
</p>


$$ g(e_1, R, e_2) = u_R^Tf \left( e_1^T W_R^{[1:k]}e_2 + V_R \begin{bmatrix} e_1 \\ e_2 \end{bmatrix} + b_R \right) $$

$f=\tanh$, $W_R^{[1:k]} \in \mathbb{R}^{d\times d\times k}$, $V_R\in \mathbb{R}^{k\times2d}$, 그리고 $u, b_R \in \mathbb{R}^k$ 입니다. bilinear form의 장점은 두 벡터를 곱셈 연산을 통해서 연관시킨다는 것입니다. 반면에, 일반적인 single layer 형태에서는 덧셈 연산을 통해 연관시킵니다.

NTN에서는 bilinear form을 하나의 slice로 보아 이들을 쌓습니다. '각각의 slice는 한 유형의 entity pair, 또는 관계 인스턴스를 생성'하는 것으로 볼 수 있습니다. 이렇게 slice를 쌓음으로써 두 벡터 간 상호작용을 좀 더 다양하게, 그리고 더 잘 이해할 수 있을 것 같습니다. 여기서 $e_1^T W_R^{[1:k]}e_2$ $+\, b_R$ 부분을 PyTorch Bilinear 레이어로 구현할 수 있겠죠?

---

<p>
<span class='center'>
<img src='/assets/res/etc/ntn2.png' width='90%' /></span>
</p>

$$ \begin{aligned}
(e_1, R, e_2) = &(\small{ \text{cat, has part, tail}} \normalsize), \\
&(\small{ \text{tiger, type of, cat}} \normalsize)
\end{aligned}$$

모델에 대해서 단순하게 생각해봅시다. 각 entity를 vector로 나타낸다고 했는데, 대규모 corpora에서 비지도 학습된 벡터로 word vector를 초기화했을 때 모델이 좀 더 개선된다고 합니다.

모델은 두 entity 간의 관계를 학습하게 됩니다. 관계 $R$은 그림의 왼쪽 블록처럼 여러 유형이 있을 수 있습니다. '고양이는 꼬리가 있다', '호랑이는 고양이과이다' 라는 관계를 알고 있을 때 '호랑이는 꼬리가 있다' 라는 지식을 추론할 수 있느냐가 평가 대상이 될 수 있겠습니다.

그리고 논문에서는 단어를 구성하는 벡터들의 average로 entity를 나타내었을 때 성능이 향상되었다고 하네요. 위 그림의 오른쪽 블록에서 `Bengal tiger`를 하나의 벡터로 보지 않고, 지역을 나타내는 `Bengal`과 `tiger`를 합쳐 $e_1$을 만들었습니다. 이 방식을 이용하면, 'Sumatran tiger'나 'Bengal tiger'처럼 entity name이 비슷한 부분 문자열을 공유할 경우 statistical strength를 공유하게 될 것입니다. 

$$J(\Omega) = \sum_{i=1}^N\sum_{c=1}^C \max \left( 0, 1 - g\left(T^{(i)}\right) + g\left(T_c^{(i)}\right) \right) + \lambda \| \Omega \|_2^2 $$

논문에서는 NTN 모델에 특별한 목적함수를 사용하여 모델을 최적화합니다. $N$개의 훈련시킬 correct triplets $T$가 있고, 각 triplet마다 $C$개의 오답 세트, random corrupted triplets $T_c$가 있습니다. 그래서 $g(T)$가 entity중 하나를 무작위로 교체한 $g(T_c)$들보다 높도록, 이상적으로는 margin인 1이 되도록 학습을 시킵니다. 이와 관련된 기법으로는 <strong><font color='#FF7D33'>Negative Sampling</font></strong> 을 참고하시길 바랍니다.

$ \lambda \| \Omega \|_2^2$ 항은 NTN 모델의 파라미터들을 $\Omega=u,W,V,b,E$라고 할 때, $L_2$ regularization 항을 추가한 것입니다. 모델에 대한 더 자세한 내용은 해당 논문을 참고해주세요.
